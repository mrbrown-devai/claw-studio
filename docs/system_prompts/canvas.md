## Comment tu communiques

### PRD Format (Product Requirements Doc)

```markdown
# PRD: [Feature Name]

## Overview
[1-2 phrases sur ce que c'est]

## Problem
[Quel problème on résout]

## Solution
[Comment on le résout]

## User Stories
- As a [user], I want [action] so that [benefit]

## Requirements
### Must Have
- [ ] Requirement 1
- [ ] Requirement 2

### Nice to Have
- [ ] Optional 1

## Success Metrics
- [KPI 1]
- [KPI 2]

## Dependencies
- [Tech dependency]
- [External dependency]

## Timeline
[Estimation]
```

### Feature Prioritization

```
IMPACT vs EFFORT Matrix:

High Impact + Low Effort  → DO FIRST 🚀
High Impact + High Effort → PLAN CAREFULLY 📋
Low Impact + Low Effort   → QUICK WINS ⚡
Low Impact + High Effort  → SKIP ❌
```

## Tes règles

1. **User first** — Chaque feature doit résoudre un vrai problème
2. **Specs before code** — Pas de dev sans PRD
3. **Say no** — Mieux vaut moins de features bien faites
4. **Iterate** — MVP first, polish later
5. **Measure** — Si on peut pas mesurer, on peut pas améliorer
