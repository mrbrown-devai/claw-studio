## Content Types

### 🧵 Threads (X/Twitter)

Format standard :
```
1/ Hook — Le problème ou la promesse
   ↓
2-8/ Body — Le contenu, les étapes
   ↓
9/ Summary — Récap rapide
   ↓
10/ CTA — Action + retweet ask
```

Règles :
- 1 tweet = 1 idée
- Visuals quand possible
- Break complex → simple

### 📝 Articles (Blog/Mirror)

Format standard :
```markdown
# Title (< 60 chars)

## TL;DR
[3 bullet points max]

## Introduction
[Hook + context]

## Body
### Section 1
### Section 2
### Section 3

## Conclusion
[Summary + CTA]
```

### 📧 Email/Announcement

Format standard :
```
Subject: [Clear benefit] — [Urgency if applicable]

Hey [name],

[1 sentence hook]

[2-3 sentences of value]

[Clear CTA button/link]

[Sign-off]
— Claw Studio
```

## Voice Guidelines

### Claw Studio Voice

| Attribute | Description |
|-----------|-------------|
| **Confident** | We know what we're doing |
| **Clear** | No jargon soup |
| **Helpful** | Value first, sell second |
| **Human** | Not robotic, not cringe |

### Do ✅

- "Here's how to..."
- "We built X because..."
- "This works because..."
- Active voice
- Concrete examples

### Don't ❌

- "In this article, we will..."
- "It is important to note that..."
- "Utilize" (use "use")
- Passive voice excessive
- Jargon without explanation
